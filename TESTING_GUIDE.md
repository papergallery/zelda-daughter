# Zelda's Daughter — Гайд по тестированию на Android-эмуляторе

Автоматизированное тестирование через Claude Code агентов на VPS.
Ориентировочное время полного прогона: **40-50 минут**.

**Инструменты:**
- `adb` — input (тап, свайп, лонг-пресс), screencap, logcat
- `android-mcp-server` — высокоуровневые команды эмулятора
- Debug-логи `[ZD:*]` — структурированный вывод из игры через logcat

**Эмулятор:** Pixel 4, экран 1080x2340, Android 11.
**Координаты:** центр (540, 1170), верхняя треть y~600, нижняя треть y~1800.

---

## 0. Перед запуском

- [ ] Эмулятор запущен: `./start_android_emulator.sh`
- [ ] APK установлен: `adb install -r path/to/ZeldaDaughter.apk`
- [ ] Пакет: `com.papergallery.zeldasdaughter`
- [ ] Activity: `com.unity3d.player.UnityPlayerActivity`
- [ ] Logcat очищен: `adb logcat -c`
- [ ] Запуск приложения:
  ```bash
  adb shell monkey -p com.papergallery.zeldasdaughter -c android.intent.category.LAUNCHER 1
  ```

## 0.5. Быстрый smoke-тест (2 мин)

Минимальный набор проверок перед полным прогоном. Если smoke не проходит — полный прогон бессмысленен.

1. **Запуск** — подождать 10 сек, сделать скриншот. Экран не розовый?
2. **Свайп** — свайп вверх, подождать 1 сек, скриншот + проверить лог `[ZD:Move]`. Персонаж двигается?
3. **Тап по объекту** — тап по ближайшему объекту, проверить лог `[ZD:Interact]`. Взаимодействие работает?
4. **Ошибки** — logcat без Exception/Error?
5. **FPS** — значение из `[ZD:Perf]` >= 30?

### Автоматизация
```bash
# 1. Запуск + скриншот
adb shell monkey -p com.papergallery.zeldasdaughter -c android.intent.category.LAUNCHER 1
sleep 10
adb exec-out screencap -p > /tmp/test_smoke_launch.png

# 2. Свайп вверх + скриншот
adb shell input swipe 540 1400 540 800 300
sleep 1
adb exec-out screencap -p > /tmp/test_smoke_move.png
adb logcat -d -s Unity | grep "\[ZD:Move\]" | tail -5

# 3. Тап по объекту
adb shell input tap 540 1000
sleep 1
adb logcat -d -s Unity | grep "\[ZD:Interact\]" | tail -5

# 4. Проверка ошибок
adb logcat -d -s Unity | grep -iE "Exception|Error|NullRef" | head -20

# 5. FPS
adb logcat -d -s Unity | grep "\[ZD:Perf\]" | tail -3
```

**Результат:** если любой из пунктов 1-4 провален — стоп, чинить билд.

## Debug-логи [ZD:*]

Все игровые системы пишут структурированные логи с тегами `[ZD:*]`. Это основной инструмент автоматической проверки.

### Теги

| Тег | Система | Примеры |
|-----|---------|---------|
| `[ZD:Input]` | Ввод | Тапы, свайпы, лонг-прессы, координаты |
| `[ZD:Move]` | Движение | Направление, скорость, позиция, замедление |
| `[ZD:Interact]` | Взаимодействие | Подбор предметов, NPC, ресурсы, осмотр |
| `[ZD:Inventory]` | Инвентарь | Добавление/удаление, крафт, вес, перегруз |
| `[ZD:Combat]` | Бой | Удары, урон, раны, лечение, нокаут |
| `[ZD:Progression]` | Прогрессия | Рост навыков, мастерство оружия |
| `[ZD:Save]` | Сохранение | Автосейв, загрузка, контрольные точки |
| `[ZD:Scene]` | Сцена | Загрузка сцен, время суток, погода |
| `[ZD:Perf]` | Производительность | FPS, память, время кадра |

### Фильтрация

```bash
# Все логи игры
adb logcat -d -s Unity | grep "\[ZD:"

# Конкретная система
adb logcat -d -s Unity | grep "\[ZD:Combat\]"
adb logcat -d -s Unity | grep "\[ZD:Move\]"
adb logcat -d -s Unity | grep "\[ZD:Inventory\]"

# Комбинация систем
adb logcat -d -s Unity | grep -E "\[ZD:(Combat|Move)\]"

# Только ошибки в конкретной системе
adb logcat -d -s Unity | grep "\[ZD:Save\]" | grep -i "error\|fail"

# Последние N записей
adb logcat -d -s Unity | grep "\[ZD:Perf\]" | tail -20

# Живой поток (для мониторинга в реальном времени)
adb logcat -s Unity | grep "\[ZD:"
```

### Сохранение полного лога

```bash
# Сохранить весь лог сессии
adb logcat -d > /tmp/logcat_full.txt

# Только игровые логи
adb logcat -d -s Unity | grep "\[ZD:" > /tmp/logcat_zd.txt
```
