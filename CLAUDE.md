# Zelda's Daughter — Мобильная Action-Adventure

## О проекте

Изометрическая action-adventure с элементами survival и RPG. Исекай: обычный человек из современного мира попадает в фэнтезийный мир без суперспособностей. Управление одной рукой, вертикальная ориентация.

Дизайн-документ: `project-design.md`

## Технический стек

- **Движок:** Unity (C#)
- **Платформа:** iOS / Android (вертикальная ориентация), возможно расширение
- **Визуальный стиль:** Low-poly 3D (прототип) → Hand-drawn (целевой)
- **Ассеты для прототипа:** Бесплатные (CC0), см. ниже

## Главный принцип разработки

**Всё, что можно сделать кодом — делается кодом, не руками в Unity Editor.**

- Сцены собираются из конфигов через Editor-скрипты, не ручным drag & drop
- Prefab-ы создаются и настраиваются через код
- Материалы, коллайдеры, компоненты назначаются скриптами
- Animator Controller генерируется кодом
- Тесты запускаются автоматически (EditMode/PlayMode через CLI)
- Билды через командную строку (batch mode)
- Unity MCP для инспекции и управления Editor из агентов
- Ручная работа в Unity — только для визуальной проверки результата

Цель: минимизировать время разработчика в Unity Editor, максимизировать автоматизацию через агентов.

## Ключевые принципы дизайна

- **Без UI** — нет HP-баров, показателей, цифр. Состояние персонажа — через визуал и реплики
- **Один палец** — всё управление через свайп (движение), тап (действие), лонг-пресс (меню). Комбинированный диспетчер жестов: лонг-пресс только на персонаже + ≥0.5 сек + ≤20px смещения; иначе — свайп
- **Kenshi-прогрессия** — навыки растут через использование, победы и неудачи. Нет очков опыта. Грузоподъёмность тренируется ходьбой с перегрузом
- **Инвентарь** — гибрид: слоты (количество) + скрытый вес (перегруз → замедление, реплики, без цифр на экране)
- **Сохранение** — бесшовный автосейв: при выходе, смене зоны, регулярно. Один слот, игрок не думает о сейвах
- **День/ночь** — полный визуальный цикл с динамическим освещением. NPC живут по расписанию, ночью опаснее, факелы нужны
- **Экономика** — бартер на старте (герой не знает валюту), быстрый переход к монетам. Оба способа работают параллельно всю игру. Торговцы ценят разные предметы по-разному
- **Раны** — четыре типа: колотые/резаные (кровотечение → бинт), переломы (шина), ожоги (мазь), отравление (антидот). Крафт из трав, рецепты от эльфов или эксперименты. Заживают сами (медленно), быстрее при отдыхе, почти мгновенно от лекарств
- **Еда/голод** — еда = слабый универсальный хил. Скрытый голод на бэкенде, реплики при голоде, деградация характеристик при пустом желудке. Есть нужно нечасто
- **Прототип** — один стартовый регион (лес/поляна + город людей). Остальные расы и биомы после валидации
- **Пролог** — пробуждение у дороги → ~15 сек ходьбы → крестьянин в поле (жесты/иконки) → направляет к городу рядом
- **Движение** — CharacterController (персонаж) + Rigidbody на окружении (кусты/трава реагируют на проход)
- **Построение мира** — ручная расстановка через AI-инструменты (Editor-скрипты, конфиги, процедурная подсыпка декора)
- **Лут** — враги оставляют тушу; минимальный лут без инструмента, полная разделка с ножом (шкура, мясо, кости)
- **Еда** — drag из инвентаря на персонажа → слабый хил + утоление голода
- **Отдых** — зона костра = автоматическое ускоренное восстановление; сон (тап по кровати) = промотка времени + лечение до порога
- **Онбординг** — неблокирующие визуальные подсказки, исчезают после выполнения действия
- **Вода** — можно зайти (замедление, по пояс), нельзя плыть, переправа по мосту
- **День/ночь длительность** — 20-30 реальных минут = полный цикл (настраивается)
- **Раны от врагов** — кабан → перелом, волк → колотая (кровотечение), огонь → ожог, яд → отравление
- **Отдых** — костёр (крафт: дрова в мир + кремень на них) = пассивный отдых; сон только в таверне
- **Кузница** — плавильня (руда→металл) + наковальня (металл+палка=оружие); оружие только на наковальне
- **NPC города** — ~7-8: торговец, кузнец, бармен, травница, стражник, 2-3 жителя
- **Торговцы** — статический инвентарь + выкуп проданного игроком
- **Музыка бардов** — бесплатные фольклорные источники для MVP
- **Мультирешения** — каждая задача имеет несколько способов решения
- **Интуитивность** — крафт через drag & drop, язык учится через общение, карта заполняется через NPC

## План разработки

Полный план из 10 этапов:

0. **Подготовка** — Unity-проект, структура папок, импорт ассетов, изометрическая камера
1. **Движение и мир** — управление одним пальцем, тайловый регион, звуки шагов и природы
2. **Взаимодействие с миром** — тап по объектам, подбор предметов, добыча ресурсов
3. **Инвентарь и крафт** — радиальное меню, drag & drop, крафтовые комбинации, размещение предметов в мире, логические цепочки
4. **Боевая система** — тап = удар, уклонение через движение, враги (кабан, волк), раны, нокаут
5. **Kenshi-прогрессия** — скрытые характеристики, рост через использование/бой/победы/неудачи
6. **NPC и язык** — город с NPC, языковая механика (кракозябры → понимание), торговля
7. **Карта, блокнот, квесты** — покупка карты, заполнение через диалоги, блокнот, псевдо-квесты
8. **Физика среды и звук** — взаимодействия стихий, диегетическая музыка (барды), звук города
9. **Обучение оружию и полировка** — обучение у рас, прогрессия точности, баланс, тестирование

### Unity-интеграция (настроить при создании проекта)
- **CoplayDev/unity-mcp** — MCP-сервер для управления Unity Editor (268 инструментов)
- **Unity batch mode** — CLI для компиляции, тестов, билдов
- **Unity Test Framework** — EditMode/PlayMode тесты

### Правила кода (C#)
- Prefer composition over inheritance
- ScriptableObject для данных, MonoBehaviour для поведения
- Namespaces: `ZeldaDaughter.{Input,Combat,Inventory,Progression,World,NPC,Save,UI}`
- Системы общаются через C# events, не прямые ссылки
- Все числовые параметры в ScriptableObject, не хардкод
- `[SerializeField] private` вместо `public` для инспектора
- Тесты обязательны для систем с формулами
