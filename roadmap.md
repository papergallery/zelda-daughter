## Этап 1: Движение и мир

> Цель: персонаж ходит по тайловому региону, управляемый одним пальцем.

### 1.1 Управление персонажем
- [ ] 🟡 Система ввода: определение свайпа (ведение пальцем) → направление движения
- [ ] 🟡 CharacterController для движения персонажа в направлении свайпа (основной контроллер)
- [ ] 🟢 Rigidbody на объектах окружения (кусты, трава) — реакция на проход персонажа (отклонение, покачивание)
- [ ] 🟢 Поворот персонажа в сторону движения (плавный)
- [ ] 🟢 Анимация idle при отсутствии ввода
- [ ] 🟢 Анимация walk/run при движении
- [ ] 🟡 Переключение walk ↔ run (по расстоянию пальца от точки начала свайпа)
- [ ] 🟢 Остановка при отпускании пальца

### 1.2 Камера
- [ ] 🟡 Плавное следование камеры за персонажем (Cinemachine или кастом)
- [ ] 🟢 Ограничение камеры границами региона (камера не уходит за пределы карты)

### 1.3 Стартовый регион (ручная расстановка через AI-инструмент)
- [ ] 🟡 Конфиг-схема региона: дорога (место пробуждения) → поле крестьянина с хижиной (~15 сек ходьбы) → город людей (рядом с полем); лес, поляна, река/ручей вокруг
- [ ] 🟡 Генерация крупных объектов через AI-инструмент: здания города, хижина, дорога, мост, река — по конфигу с координатами
- [ ] 🟡 Процедурная подсыпка декора: деревья, камни, трава, кусты, цветы — через скрипт по зонам с контролем плотности
- [ ] 🟢 Настроить коллайдеры на препятствиях (деревья, скалы, вода, здания)
- [ ] 🟢 NavMesh для проходимости
- [ ] 🟡 Вода: персонаж может зайти в воду (замедление, визуал по пояс), но не может плыть — на глубине невидимая стена; переправа через реку — по мосту

### 1.4 Цикл дня и ночи
- [ ] 🟡 Внутриигровые часы: таймер, определяющий текущее время суток (день/закат/ночь/рассвет); полный цикл = 20-30 реальных минут (настраиваемый параметр)
- [ ] 🟡 Динамическое освещение: вращение Directional Light (солнце → луна), смена цветовой температуры и интенсивности через Gradient/AnimationCurve
- [ ] 🟡 Skybox / фон: плавная смена цвета неба в зависимости от времени суток
- [ ] 🟢 Тени: настройка теней от динамического света, корректная работа на мобильных (shadow distance, resolution)
- [ ] 🟢 Ночная видимость: снижение дальности видимости ночью, fog или post-processing

### 1.5 Звуки окружения
- [ ] 🟡 Система звуков шагов по разным поверхностям (трава, камень, грязь, дерево)
- [ ] 🟢 Ambient-звуки природы (птицы, ветер, вода) — зонированные AudioSource
- [ ] 🟢 Миксер: баланс громкости шагов и окружения
- [ ] 🟢 Разные ambient-звуки для дня и ночи (ночью — сверчки, совы; днём — птицы)

### 1.6 Онбординг (визуальные подсказки)
- [ ] 🟡 Система неблокирующих подсказок: полупрозрачные визуальные хинты на экране; не блокируют ввод; каждая подсказка исчезает только после выполнения действия
- [ ] 🟢 Подсказка движения: иконка свайпа, появляется при пробуждении, исчезает после первой ходьбы
- [ ] 🟢 Подсказка тапа: иконка тапа по объекту, появляется при приближении к первому интерактивному объекту, исчезает после первого тапа
- [ ] 🟢 Подсказка инвентаря: иконка лонг-пресса на персонаже, появляется после первого подбора предмета, исчезает после первого открытия меню

## Этап 2: Взаимодействие с миром

> Цель: тап по объектам вызывает взаимодействие — подбор, добыча, осмотр.

### 2.1 Система тапа
- [ ] 🟡 Raycast от тапа по экрану в мир (через камеру)
- [ ] 🟡 Определение типа объекта под тапом (интерактивный / неинтерактивный)
- [ ] 🟢 Персонаж подходит к объекту перед взаимодействием (pathfinding к точке)
- [ ] 🟢 Визуальная подсветка / outline интерактивных объектов при приближении

### 2.2 Подбор предметов
- [ ] 🟡 Компонент `Pickupable` — предмет можно подобрать тапом
- [ ] 🟢 Анимация подбора (наклон, подъём руки)
- [ ] 🟢 Предмет исчезает из мира → добавляется в инвентарь (пока простой список)
- [ ] 🟢 Реплика персонажа при подборе («Палка... сгодится», «О, камень»)

### 2.3 Добыча ресурсов
- [ ] 🟡 Система добычи: тап по дереву/камню/кусту → анимация удара → выпадает ресурс
- [ ] 🟢 Разные ресурсы из разных объектов (дерево → палка/дрова, камень → камень, куст → ягоды/ветки)
- [ ] 🟢 Прочность объектов (дерево = 5 ударов палкой, 2 удара топором)
- [ ] 🟢 Визуальная деградация при добыче (дерево качается, камень трескается)
- [ ] 🟡 Респаун ресурсов через время

### 2.4 Базовое NPC-взаимодействие (пролог)
- [ ] 🟡 Компонент `NPCInteractable`: тап по NPC → персонаж подходит → NPC поворачивается к игроку
- [ ] 🟡 Общение жестами/иконками: NPC показывает иконки над головой (направление-стрелка, предмет, эмоция); набор из 10-15 базовых пиктограмм
- [ ] 🟢 Крестьянин-пролог: при первом тапе — жест приветствия → иконка-стрелка в сторону города
- [ ] 🟢 Ответы игрока иконками (упрощённые): выбор из 2-3 пиктограмм для базового взаимодействия

### 2.5 Осмотр
- [ ] 🟢 Тап по необычному объекту → реплика персонажа с описанием
- [ ] 🟢 Система реплик персонажа (текстовое облачко над головой, таймер исчезания)

### 2.6 Система сохранения
- [ ] 🟡 Сериализация состояния мира: позиция персонажа, подобранные предметы, состояние ресурсов (срубленные деревья и т.д.)
- [ ] 🟡 Автосейв: сохранение при выходе из приложения (OnApplicationPause/OnApplicationQuit), при смене зоны, по таймеру (каждые N минут)
- [ ] 🟡 Загрузка: при старте приложения — восстановление состояния, игрок на том же месте
- [ ] 🟢 Один слот сохранения (JSON или бинарный файл в Application.persistentDataPath)

**Итого: ~23 задачи**

## Этап 3: Инвентарь и крафт

> Цель: радиальное меню, drag & drop крафт, размещение предметов в мире.

### 3.1 Радиальное меню
- [ ] 🟡 **Диспетчер жестов:** единая точка входа для всех касаний — определяет тип жеста (тап / свайп / лонг-пресс) по времени удержания, смещению пальца и зоне начала касания
- [ ] 🟡 **Детекция лонг-пресса (комбинированная):** срабатывает ТОЛЬКО если: (1) касание началось в зоне персонажа (экранная проекция коллайдера), (2) удержание ≥0.5 сек, (3) смещение пальца ≤20px. Если палец сдвинулся раньше — даже с персонажа — считается свайпом → движение
- [ ] 🟡 **UI радиального меню:** Canvas с секторами по кругу, расположение вокруг персонажа; палец не отрывается — ведёт к нужному сектору и отпускает
- [ ] 🟢 Визуальная обратная связь при удержании (кольцо-таймер вокруг персонажа, заполняется за 0.5 сек)
- [ ] 🟡 Секции меню: инвентарь, карта (заглушка), блокнот (заглушка)
- [ ] 🟢 Анимация открытия/закрытия
- [ ] 🟢 Пауза игры при открытом меню (или замедление)

### 3.2 Инвентарь
- [ ] 🟡 UI сетки предметов (иконки, стаки)
- [ ] 🟡 Система данных предметов (ScriptableObject: название, иконка, тип, стакается ли, максимальный стак)
- [ ] 🟢 Просмотр информации о предмете (тап → название + описание)
- [ ] 🟡 Лимит слотов: фиксированное количество ячеек, предметы стакаются по типу
- [ ] 🟡 Система веса: скрытый вес каждого предмета (в ScriptableObject), суммарный вес на бэкенде, порог грузоподъёмности
- [ ] 🟢 Штрафы за перегруз: замедление движения и атак, реплики персонажа о тяжести (привязка к порогу веса)

### 3.3 Крафт (drag & drop)
- [ ] 🟡 **Drag-система в инвентаре:** начало перетаскивания по лонг-прессу на иконке предмета (~0.3 сек); предмет «отлипает» от слота и следует за пальцем
- [ ] 🟡 **Зона дропа:** определение целевого слота при отпускании — raycast по UI, подсветка слота-кандидата при наведении
- [ ] 🟢 **Отмена перетаскивания:** если палец ушёл за пределы сетки или вернулся в исходный слот — предмет возвращается на место
- [ ] 🟡 База рецептов (ScriptableObject: ингредиент A + ингредиент B = результат)
- [ ] 🟡 Проверка комбинации при дропе → создание нового предмета
- [ ] 🟢 Анимация/эффект успешного крафта
- [ ] 🟢 Реплика персонажа при неудачной комбинации («Это не сработает»)
- [ ] 🟡 Стартовый набор рецептов полевого крафта: палка+ткань=факел, нож+палка=заострённая палка, топор+дрова=доски, ткань+травы=бинт, палка+ткань=шина, особая трава+жир=мазь от ожогов, специальные травы=антидот
- [ ] 🟡 Крафт костра (особый): игрок размещает дрова в мире → переносит кремень на дрова (drag из инвентаря на объект в мире) → костёр загорается → становится зоной отдыха + источник света

### 3.4 Станочный крафт (кузница)
- [ ] 🟡 Тап по станку → открытие UI станка (инвентарь станка + слоты материалов)
- [ ] 🟡 Перекладывание материалов из инвентаря на станок
- [ ] 🟢 Жест удара по станку → результат (если рецепт валиден)
- [ ] 🟡 Плавильня: руда → расплавленный металл (положить руду, подождать, забрать результат)
- [ ] 🟡 Наковальня: металл+палка=меч, металл+короткая палка=нож, металл=наконечник стрелы; оружие крафтится ТОЛЬКО здесь
- [ ] 🟢 Станки в мире: плавильня и наковальня у кузнеца в городе

### 3.5 Размещение предметов в мире
- [ ] 🟡 **Детекция drag-за-край:** при перетаскивании предмета отслеживать выход пальца за границу панели инвентаря; при пересечении границы — переход в режим «размещение в мире»
- [ ] 🟡 **Переход UI → мир:** закрытие инвентаря с анимацией, создание «призрака» предмета в мировом пространстве под пальцем (полупрозрачная модель, следует за касанием через Raycast на terrain)
- [ ] 🟢 **Подтверждение размещения:** отпускание пальца = ставим предмет; если позиция невалидна — призрак красный, при отпускании предмет возвращается в инвентарь
- [ ] 🟡 Проверка валидности позиции для размещения (не на стене, не в воздухе)
- [ ] 🟢 Реплика при невалидном месте
- [ ] 🟢 Предмет остаётся в мире как интерактивный объект

### 3.6 Логические цепочки
- [ ] 🟡 Поднесение предмета к объекту в мире (факел к костру → факел загорается)
- [ ] 🟢 Подсказки персонажа: при удержании предмета рядом с подходящим объектом → реплика-намёк

**Итого: ~33 задачи**

## Этап 4: Боевая система

> Цель: тап по врагу = удар, уклонение через движение, раны, нокаут.

### 4.1 Базовый бой
- [ ] 🟡 Тап по врагу → персонаж подходит и наносит удар
- [ ] 🟡 Каждый тап = один удар (без автоатаки)
- [ ] 🟡 Анимации атаки (палка, меч, кулак — минимум 3 типа оружия)
- [ ] 🟢 Хитбокс/триггер удара → нанесение урона врагу

### 4.2 Враги
- [ ] 🟡 AI кабана: бродит → агрится при атаке → бежит на игрока → удар головой → наносит перелом
- [ ] 🟡 AI волка: патрулирует → замечает игрока → подкрадывается → бросок/укус → наносит колотую рану (кровотечение)
- [ ] 🟡 Общий FSM врагов (Idle → Alert → Chase → Attack → Stagger → Death)
- [ ] 🟢 Анимация замаха перед атакой (визуальный сигнал игроку)
- [ ] 🟢 Окно для уклонения (игрок отводит палец, персонаж отходит)
- [ ] 🟢 Анимации смерти/убегания врагов
- [ ] 🟡 Спавн и респаун врагов в зонах
- [ ] 🟡 Лут с врагов: туша остаётся в мире после смерти; без инструмента — минимальный лут (клык, перо); с разделочным ножом — полная разделка (шкура, мясо, кости); тап по туше с ножом в инвентаре → анимация разделки → ресурсы в инвентарь

### 4.3 Урон и состояние персонажа
- [ ] 🟡 Базовая модель здоровья: скрытый HP + массив активных ран (тип, тяжесть, таймер)
- [ ] 🟡 Колотые/резаные раны: кровотечение (медленная потеря HP); визуал — кровь на одежде, прижимает руку; лечение — бинт
- [ ] 🟡 Переломы: замедление движения и атак; визуал — хромота; лечение — шина
- [ ] 🟡 Ожоги: снижение точности атак; визуал — обожжённая одежда, дует на руки; лечение — мазь
- [ ] 🟡 Отравление: периодическая тошнота (остановки), деградация всех действий; визуал — зеленоватый оттенок, держится за живот; лечение — антидот
- [ ] 🟢 Реплики персонажа о состоянии («Ничего страшного» → «Плохо дело...» → «Не могу дальше...»)
- [ ] 🟡 Изменение скорости/анимаций при ранениях
- [ ] 🟡 Естественное заживление: раны медленно проходят сами со временем
- [ ] 🟡 Отдых у костра: при нахождении в зоне костра — статус «отдых», увеличенная скорость восстановления (proximity trigger, без действий игрока); в дикой природе — единственный способ отдыха
- [ ] 🟡 Сон (только в городе/таверне): тап по кровати → затемнение (глаза закрываются) → промотка времени → пробуждение; восстановление ран до порога; крафт спальников НЕ предусмотрен
- [ ] 🟡 Скрытый параметр голода: реплики при голоде, деградация характеристик за порогом; есть нужно нечасто
- [ ] 🟡 Механика еды: drag еды из инвентаря на персонажа → анимация поедания → слабый универсальный хил + утоление голода

### 4.4 Нокаут
- [ ] 🟡 При критическом уроне → нокаут
- [ ] 🟡 Визуал нокаута: затемнение с проблесками → пробуждение
- [ ] 🟢 После нокаута: раны остаются, персонаж встаёт с ранениями
- [ ] 🟢 Короткие нокауты в ранней игре (без раздражения)

### 4.5 Оружие
- [ ] 🟡 Система экипировки оружия (слот в инвентаре → в руке персонажа)
- [ ] 🟢 Разный урон/скорость от разного оружия (палка, меч, молот)
- [ ] 🟢 Визуальное отображение оружия в руке модели

**Итого: ~31 задача**

## Этап 5: Kenshi-прогрессия

> Цель: скрытые характеристики, растущие через использование и бой.

### 5.1 Система характеристик
- [ ] 🟡 Скрытые параметры персонажа: сила, стойкость, ловкость, точность, выносливость, грузоподъёмность
- [ ] 🟡 Формулы влияния параметров на геймплей (урон, скорость атаки, шанс промаха, устойчивость к урону)
- [ ] 🟢 Сохранение/загрузка параметров

### 5.2 Рост через использование
- [ ] 🟡 Трекинг действий: каждый удар, промах, полученный урон, пройденное расстояние, время ходьбы с перегрузом
